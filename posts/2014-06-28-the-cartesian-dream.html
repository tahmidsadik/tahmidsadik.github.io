The Cartesian Dream or Higher Order Functions

I was always fascinated by the use of higher order functions. They are small, elegant, consists of core building blocks of any programming language, functions. Now, functions are simple. We all know about functions, so let's focus on the difference between them.

  • Higher order functions may ***take other functions*** as input.
  • Higher order functions may ***return other functions*** as output.

Let's write a function that will return another function ( just think about it for a second, You are generating a function dynamically, whoa ).

Here's an example in clojure

(defn multiple-of-x? [x]
  (fn [n] (zero? (rem n x))))

((multiple-of-x? 3) 15)         ;; true
((multiple-of-x? 5) 15)         ;; true
((multiple-of-x? 9) 15)         ;; false
The function multiple-of-x? takes a number x as input and returns a functionwhich itself takes another number n as input and returns true if n isa multiple of x, otherwise false.

Writing higher order functions might take some time, but they also tend to become much more useful. They also force you to think about generalising a problem and think through, which in turn might make you a better programmer.
